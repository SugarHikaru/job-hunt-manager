# 要件定義

## １．概要

### １－１．アプリ名
就活マネージャー

### １－２．開発目的
就活活動における企業情報、ES 締切、面接日程などを一元管理し、
就活状況を効率的に把握できるようにする。

### １－３．背景
複数企業の先行管理を手作業で行うと、
締切漏れや情報整理が困難になるため。

### １－４．想定利用者
開発者本人（大学生）

--------------------------------------------------------------------------

"" ２．業務要件（システム要件）

## ２－１．現状の課題
- ES 締切を忘れる
- 面接日程が整理しずらい
- 企業情報が分散する
- マイページ ID を管理しずらい

### ２－２．システム化する内容
- 企業情報管理
- 選考情報管理
- 面接日程管理
- ES 締切管理

### ２－３．システム化しない内容
- ES 本文作成
- 面接練習

---------------------------------------------------------------------

## ３．機能要件

### ３－１．ログイン機能
- ID/PW 認証

### ３－２．企業管理機能
- 企業追加
- 編集
- 削除
- 一覧表示

### ３－３．先行管理機能
- 選考状況登録
- ステータス更新

### ３－４．スケジュール管理
- 面接日程登録
- ES 締切表示


------------------------------------------------------------------------

## ４．非機能要件

### ４－１．パフォーマンス
- 一覧画面を3秒以内に表示

### ４－２．セキュリティ
- ログイン認証を実装
- パスワードを暗号化

### ４－３．保守性
- Spring Boot の MVC 構成を採用

### ４－４．開発環境
- Java
-Spring Boot
- MySQL
- VACode
